# سامانه بهره‌وری هوشمند — Hospital Productivity AI Dashboard (MVP)

نمونهٔ تعاملی (Visual MVP) برای ارائه به سرمایه‌گذار — یک پلتفرم **داشبوردساز هوشمند مبتنی بر هوش مصنوعی** برای پایش بهره‌وری بیمارستان.
کل برنامه **راست‌چین (RTL)**، **فارسی**، با تم **تیره و enterprise** و فونت **Vazirmatn** ساخته شده است.

> همهٔ رفتار «هوش مصنوعی» **شبیه‌سازی‌شده و هاردکد** است — هیچ تماس واقعی با مدل/بک‌اند انجام نمی‌شود. این یک دموی کلیک‌محور برای پیچ‌دک است.

---

## اجرا (Run)

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # تولید نسخهٔ production در dist/
npm run preview    # سرو نسخهٔ build
```

نیازمند Node ≥ ۱۸.

## استک فنی

| لایه | انتخاب |
|------|--------|
| Framework | React 18 + TypeScript + Vite |
| Styling | Tailwind CSS (فقط utilityهای منطقی `ms/me/ps/pe/start/end` برای RTL) |
| UI primitives | کامپوننت‌های دست‌ساز به سبک **shadcn/ui** (Card / Button / Badge / StatusDot) |
| Charts | **Recharts** (همه با محور معکوس RTL و ارقام فارسی) |
| Animation | Framer Motion |
| Icons | lucide-react (بدون اموجی به‌عنوان آیکون) |
| Routing/State | react-router-dom + یک Context store سبک |

تمام اعداد از طریق `toFa()` در مرز رندر (شامل داخل نمودارها، tickها و tooltipها) به ارقام فارسی تبدیل می‌شوند.

## دو جریان قهرمان (Hero Flows)

### داشبوردساز (صفحهٔ ۲)
سه جفت **پرامپت → نتیجهٔ** از پیش‌نوشته (در `src/data/mock.ts` → `BUILDER_SCRIPTS`):

| پرامپت | خروجی |
|--------|-------|
| تعداد بیماران فعلی در اتاق عمل | کارت عدد تکی + اسپارک‌لاین |
| جراحانی که بیشترین عمل را در ۵ ماه اخیر داشته‌اند | نمودار میله‌ای افقی (Top ۵) |
| روند درآمد اورژانس در ۶ ماه گذشته | نمودار خطی + اختلاف دوره |

### تحلیل‌یار (صفحهٔ ۶)
سه حالت روی یک صفحه: **پرسش → تفکر → نتیجه**. تایم‌لاین ۶ مرحله‌ای با reveal میانی (چیپ شاخص‌ها، اسپارک‌لاین چندخطی، نمودار وزن‌دهی، فهرست رتبه‌بندی) و سپس حکم «شفا یحیائیان با امتیاز ۸۱…» + عوامل مؤثر + راهکارها + بازشوندهٔ «تحلیل ریشه‌ای ۵ لایه». اسکریپت در `src/data/analyst.ts`.

موتور زمان‌بندی مشترک: `src/lib/useTimedSteps.ts` — همهٔ تایمرها هنگام unmount/restart پاک می‌شوند تا مراحل روی صفحهٔ اشتباه اجرا نشوند.

## مسیر پیشنهادی دمو (Investor Click-Path)

1. **خانه** → روی چیپ «جراحانی که بیشترین عمل…» بزن → به **داشبوردساز** می‌رود و تولید خودکار شروع می‌شود → نمودار میله‌ای ساخته می‌شود → «ذخیره داشبورد» (در گالری خانه ظاهر می‌شود).
2. **لحظه‌نما** → نبض زندهٔ بیمارستان.
3. **دیده‌بان** → سلول‌های بحرانی قرمز که نیاز به اقدام دارند.
4. **ژرف‌نما** → drill-down دانشگاه ← بیمارستان ← بخش.
5. **اسناد** → «دانش هوش مصنوعی از این استانداردهای نمایه‌شده می‌آید.»
6. **تحلیل‌یار** → بپرس «کدام بیمارستان بهترین خدمات را داشته؟» → تحلیل ۶ مرحله‌ای → حکم + راهکارها. (دکمهٔ «ساخت داشبورد از این تحلیل» دوباره به داشبوردساز برمی‌گردد.)

اتصال بین صفحات از طریق `src/store/dashboards.tsx` (pendingPrompt / pendingWidget / saveDashboard) انجام می‌شود.
